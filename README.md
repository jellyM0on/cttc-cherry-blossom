### 🌸 Overview
This project was made for the exercise `Individual Activity: Data Analysis Script & Report`

It aims to explore the following research questions:
- Are cherry blossom full bloom dates occurring earlier over time in Japan?
- What is the typical timing of cherry blossom full bloom?
- Which years recorded the earliest and latest full bloom dates?
- How consistent is bloom timing, and how has this variability changed over time?
- Do bloom dates differ across Koppen climate classifications?
- Do different climate classifications exhibit different trends in bloom timing over time?
- Do bloom timing and trends differ across provinces in Japan?

Results can be seen in `outputs/`

### Data Sources
Cherry Blossom Event Data:
- [Japan Cherry Blossom Observation](https://www.kaggle.com/datasets/hideos/japan-cherry-blossom-observation)

WMO Station Location Data:
- [WMO Station List (via JMA)](https://www.kaggle.com/datasets/hideos/wmo-station-list-via-jma)
- [Nominatim API](https://nominatim.openstreetmap.org/search)

Location Climate Data:
- [Open Meteo API](https://archive-api.open-meteo.com/v1/archive)
- Meteostat via `meteostat` package
- Koppen climate classification was manually derived from the above sources
  
### Notes
- Pipeline:
  - The initial `cherry_blossom_cleaned.csv` is generated by cleaning and merging the raw files: `blossom.csv` and `stations.csv`
  - Geolocation data is fetched through Nominatim API to get fields such as `province`
  - Climate classification is derived from either the Open Meteo API or Meteostat (if rate limits by the API is encountered). These sources return historical weather data
  - The initial cleaned dataset is merged and enriched with the additional geolocation and climate data
  - Visualization and analysis
